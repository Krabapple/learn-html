TIL that:

  1. Do the markup first. Consider the CSS later. Don't think of them at the same time.
  2. When styling, first style the wrapper.
    In this case we don't have a wrapper, so style <body>.
  3. "margin: 20px auto;" means 20px margin on the top and bottom and auto on left and right.
  4. Syntax: "margin: top right bottom left"
    "margin: 20px 0 20px 0;" = "margin: 20px 0" = 20 for top/bottom and 0 for left/right
    "margin: 20px 5px 10px;" = "margin: 20px 5px 10px 5px;"
  5. <div> has no semantic meaning.
  6. Line height 1em = the height of a letter m.
  7. Check the CSS and see how we deal with ".meta xxx:after".
  8. Again, why use class not ID? Because we will repeat them over and over again in the blog, so we need to group them, instead of dealing with them one by one.
  9. While styling in CSS, if we define something twice, we will always override anything above - that's how CSS works.
    The lower it lies in the page, the more likely it will take precedence.
    However if we have "post_info a" at the top and "a" at the bottom, "post_info" takes precedence because it has more "weight", or more specific.
  10. Shrink the box by floating it to the left.

  11. Don't be more specific than you need to be.
